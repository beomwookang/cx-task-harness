# Background — 이 프로젝트의 동기와 맥락

> 이 문서는 포트폴리오용 내부 문서입니다. 오픈소스 리포지토리에는 포함하지 않습니다.

---

## 1. 채널톡과 ALF

채널톡은 채널코퍼레이션이 운영하는 올인원 AI 비즈니스 메신저입니다. 핵심 철학은 **"고객이 답이다"** — 제품도 비즈니스도 해답은 고객에게 있다는 믿음으로, 개발자·디자이너·대표까지 고객과 직접 대화합니다.

ALF는 채널톡의 AI 상담 에이전트입니다. 단순 FAQ 응답을 넘어, 주문 취소·교환·반품 등 실제 업무를 자동 처리하는 에이전트로 진화했습니다 (ALF v2, 2025.11). 누적 2,000+ 고객사가 도입했으며, 최대 80% 상담 해결률을 달성하고 있습니다.

최시원 대표의 철학: "AI 상담은 일자리를 대체하는 기술이 아니라, 상담의 질을 높이는 도구. 단순 문의는 AI가 담당하고, 사람은 공감이 필요한 영역에 집중."

---

## 2. FDE (Forward Deployed Engineer) 팀

팔란티어에서 영감을 받은 직군으로, **"고객을 더 많이 만나는 개발팀"**입니다.

핵심 프로세스: **문제 관찰 → 문제 공감 → 문제 정의 → 문제 해결 → 결과 확산**

FDE팀이 풀고 있는 두 가지 문제:
1. **ALF 확산** — ALF는 강력하지만 세팅이 어려움. 모든 고객사를 직접 세팅할 수 없으므로, 누구나 사용할 수 있는 도구를 만듦
2. **사내 AX (AI Transformation)** — 반복 업무를 AI로 자동화

FDE팀이 이미 만든 것:
- 고객 상담 데이터로 ALF 도입 리포트를 생성하는 도구
- ALF 태스크를 빠르게 만들 수 있는 도구
- 채널톡 기능을 세팅할 수 있는 도구

## 3. ALF 태스크 구조 (공식 문서 기반)

ALF 태스크는 GUI 에디터에서만 생성 가능하며, 프로그래밍 API는 공개되지 않음.

구성요소:
- **트리거**: 자연어 조건 + 고객/상담 필터
- **에이전트 단계**: 자연어 지시문 (역할, 대화 흐름, 종료 조건, 예외 사항)
- **코드 단계**: JavaScript (API 호출, 데이터 검증)
- **메시지 보내기 단계**: 고정 문구 발송
- **상담 처리 액션 단계**: 팀 배정, 태그 설정 등
- **함수 단계**: 외부 API 통신 (카페24 등 기본 제공)
- **브라우저 자동화 단계**: 웹 자동화
- **분기**: 조건 브랜치, 실패 시 분기
- **메모리**: 태스크 실행 중 임시 변수

업종별 유스케이스 (공식 문서):
| 업종 | 태스크 예시 |
|------|-----------|
| 이커머스 (저관여) | 주문 취소, 배송지 변경, 교환/환불, 사이즈/재고 확인 |
| 이커머스 (고관여) | 배송 일정 지정, AS 접수/유상 수리 분기 |
| 여행/항공/숙박 | 예약 변경, 예약 취소, 쿠폰 적용 문의 |
| 구독 서비스/SaaS | 구독일 변경, 구독 현황 조회, 정기 결제 취소 및 환불 |
| 병원/의료 | 예약 변경, 취소, 서류 발급 요청 |
| 교육/에듀테크 | 수강 등록/변경/취소, 교재 배송, 수업 일정 변경 |
| 모빌리티 | 예약 차량 변경/연장/취소, 과금 정책 안내 |

## 4. 이 프로젝트와의 관계

### CX Task Spec ↔ ALF 태스크 매핑

| ALF 태스크 구성요소 | CX Task Spec 대응 | n8n 대응 노드 |
|---|---|---|
| 트리거 (자연어 + 필터) | `trigger` | Webhook / Chat Trigger |
| 에이전트 단계 | `step.type: "agent"` | AI Agent (LangChain) |
| 코드 단계 | `step.type: "code"` | Code Node |
| 메시지 보내기 단계 | `step.type: "message"` | Send Message |
| 상담 처리 액션 단계 | `step.type: "action"` | Set / HTTP Request |
| 함수 단계 | `step.type: "function"` | HTTP Request |
| 브라우저 자동화 단계 | `step.type: "browser"` | (커스텀) |
| 분기 (브랜치) | `step.type: "branch"` | IF / Switch |
| 메모리 (임시 변수) | `memory[]` | Set Node / Variables |

### 포트폴리오 핵심 메시지

> CX Task Spec은 채널톡 ALF 태스크의 구조를 참조하여 설계한 범용 스키마입니다.
>
> - 에이전트 노드의 지시문 구조 (역할, 대화 흐름, 종료 조건, 예외 사항)를 그대로 반영
> - ALF 태스크의 모든 구성요소와 1:1 매핑
> - ALF 태스크 에디터로의 변환은 어댑터 레이어 하나만 추가하면 구현 가능
> - n8n 변환을 통해 플랫폼 접근 없이도 태스크 설계를 시각화/테스트 가능
>
> **FDE팀에 제공하는 가치:**
> - 고객사 상담 데이터 → 태스크 설계까지의 시간 단축
> - 고객사 미팅 전 n8n으로 프로토타입 시연 가능
> - 업종별 템플릿으로 레시피 재사용성 확보
